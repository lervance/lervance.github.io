# View Tracking Setup

This project now tracks website visits in two steps:

1. Increment total views via an external analytics API (`counterapi.dev`).
2. Store each visit in a Supabase database table.

## 1) Create Supabase table

Run this SQL in your Supabase SQL editor:

```sql
create table if not exists public.website_views (
  id bigint generated by default as identity primary key,
  visitor_id text not null,
  page_path text not null,
  page_url text not null,
  user_agent text,
  view_count bigint not null,
  viewed_at timestamptz not null default now()
);

alter table public.website_views enable row level security;

create policy "Allow insert website views"
on public.website_views
for insert
to anon
with check (true);
```

## 2) Configure environment variables

Create a local `.env` file from `.env.example` and fill it:

- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`
- `VITE_SUPABASE_VISITS_TABLE` (optional, defaults to `website_views`)

## 3) Deploy variables

If you deploy on GitHub Pages with a CI workflow, add the same values as repository secrets and inject them during build.
